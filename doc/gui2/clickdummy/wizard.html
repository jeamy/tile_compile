Layout 1920 Ablauf Wizard

Detaillierter Wizard-Ablauf: Input → Run-Ziel (runs_dir, run_name) → Queue → Preset/Szenario → Validation → Start → Resume mit Config-Revision.

1. Input + Run-Ziel

`input_dirs`, `runs_dir`, `run_name`; Preview zeigt `_`.

2. MONO Queue

Filterliste (`L/R/G/B/Ha...`) mit je `input_dir`, optional `pattern`, seriell `Filter i/N`.

3. Preset + Parameter

Preset/Situation anwenden, alle Felder editieren, Explain + YAML-Diff pruefen.

4. Validation

Schema+Semantik validieren; bei `error` bleibt Start/Resume blockiert.

5. Start + Monitor

Queue starten, Live-Phasen verfolgen, Stats im Run Monitor ausloesen.

6. Resume + Revision

Phase anklicken, Config-Revision waehlen, bei Bedarf alte Revision wiederherstellen (`Restore Revision`).

Implementierungsabbildung: `wizard_init -> scan_preview -> build_queue -> apply_preset/apply_scenarios -> validate_config -> persist_config_revision -> resolve_run_output_dir -> start_serial_queue`.
